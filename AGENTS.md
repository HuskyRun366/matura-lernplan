# Änderungsprotokoll (changes.md)

Jede Änderung am Code, an Konfiguration oder an Daten muss in `changes.md` im
Repo-Root dokumentiert werden — neueste Einträge oben, Format
`## YYYY-MM-DD — <Kurztitel>` gefolgt von Bullet-Liste (Was / Warum).

`changes.md` ist **immer** vor Abschluss einer Aufgabe/eines Commits zu
aktualisieren. Fehlt der Eintrag, gilt die Aufgabe als unvollständig.
